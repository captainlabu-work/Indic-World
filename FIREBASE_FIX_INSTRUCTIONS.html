Step 1: Open Firebase Console

Step 2: Navigate to Storage Rules

In the Firebase Console:

  1. Click on "Storage" in the left sidebar
  2. Click on the "Rules" tab at the top

Step 3: Replace the Rules

Copy this entire code block and paste it in the rules editor:

rules_version = '2'; service firebase.storage { match /b/{bucket}/o { // Profile photos - users can only read/write their own photo match /profile-photos/{userId} { allow read: if true; // Anyone can view profile photos allow write: if request.auth != null && request.auth.uid == userId; } // Article images - authenticated users can upload match /article-images/{allPaths=**} { allow read: if true; // Anyone can view article images allow write: if request.auth != null; } // Articles path - authenticated users can upload match /articles/{allPaths=**} { allow read: if true; // Anyone can view article images allow write: if request.auth != null; } // Avatars path - users can upload their own avatar match /avatars/{userId}/{allPaths=**} { allow read: if true; allow write: if request.auth != null && request.auth.uid == userId; } // Default deny for everything else match /{allPaths=**} { allow read, write: if false; } } }

Step 4: Publish the Rules

Click the "Publish" button to save and deploy the new rules.

Wait for the confirmation message that says "Rules published successfully".

Step 5: Test the Fix

  1. Clear your browser cache (Ctrl+Shift+Delete)
  2. Go back to your Indic website
  3. Log in if you're not already logged in
  4. Try creating and submitting a story again
